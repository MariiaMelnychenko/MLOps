# Telco Customer Churn

Проєкт для експериментів з MLOps-підходами: побудова конвеєру навчання ML‑моделей, керування експериментами, відстеження метрик та артефактів (через MLflow), а також підготовка до подальшого продакшен‑деплою.

### Можливості

- **Керування експериментами**: логування метрик, параметрів та артефактів за допомогою MLflow (`mlruns` каталог).
- **Відтворюваність**: фіксація версій залежностей та параметрів навчання.
- **Автоматизація**: можливість винесення навчання / оцінки моделі в скрипти або пайплайни CI/CD.
- **Масштабування**: підготовка до запуску навчання на віддалених середовищах або в контейнерах.

### Структура проєкту (загальна ідея)

- **`data/`**: сирі та оброблені дані (необов’язково в репозиторії).
- **`src/`**: вихідний код:
  - **`data/`** – завантаження та попередня обробка даних;
  - **`models/`** – визначення моделей та навчання;
  - **`evaluation/`** – функції оцінки якості.
- **`mlruns/`**: експерименти MLflow (метрики, параметри, моделі, артефакти).
- **`notebooks/`**: Jupyter‑ноутбуки для дослідження.
- **`venv/`**: віртуальне середовище Python (зазвичай не додається в git).

> Примітка: фактична структура каталогу може відрізнятися – онови опис під свій проєкт.

### Вимоги

- **Python**: 3.10+ (рекомендовано)
- **ОС**: Windows 10+ / Linux / macOS
- **Основні бібліотеки** (приклад):
  - `mlflow`
  - `scikit-learn`
  - `pandas`
  - `numpy`
  - `matplotlib` або `seaborn`

Якщо у тебе є `requirements.txt` або `pyproject.toml`, використовуй їх як джерело істини для залежностей.

### Встановлення

1. **Клонувати репозиторій**:

   ```bash
   git clone <URL_ТВІЙ_РЕПОЗИТОРІЙ>
   cd mlops1
   ```

2. **Створити та активувати віртуальне середовище (рекомендовано)**:

   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```

3. **Встановити залежності**:

   ```bash
   pip install -r requirements.txt
   ```

   або, якщо використовуєш Poetry / інший менеджер, – адаптуй команду відповідно.

### Запуск експериментів

Нижче приклад типового сценарію (скорегуй під свої файли):

- **Навчання моделі**:

  ```bash
  python src/train.py
  ```

- **Оцінка моделі**:

  ```bash
  python src/evaluate.py
  ```

- **Запуск MLflow UI** для перегляду експериментів:

  ```bash
  mlflow ui --backend-store-uri mlruns
  ```

Після цього інтерфейс буде доступний у браузері за адресою `http://127.0.0.1:5000` (за замовчуванням).
